写一个开源项目，目标是通过 GitHub Pages 构建一个可交互的静态电影评分网站，聚合 RottenTomatoes（烂番茄）和豆瓣评分，解决国内用户无法直接访问烂番茄的问题。

爬取烂番茄网站数据：获取电影信息、影评人评分、观众评分等
GitHub 开源：为开源运动贡献
GitHub Pages 静态网站：生成可交互的纯静态网站，解决国内访问烂番茄问题
豆瓣链接匹配：爬取豆瓣相关电影链接，快速定位国内平台
定时爬取：自动定期爬取数据，自动更新网站
电影分享模块：自动生成分享图片
加权计算：对烂番茄影评人评分 + 观众评分 + 豆瓣评分进行加权，按加权分排序
Headers 请求头：增加请求头反检测
要求提出更多改进想法，并优化代码问题

数据库错误日志、图片下载指数退避重试

修复各种 bug：str/int 类型错误、数据库列缺失、safe_extract 参数错误、JS 语法错误

初始化浏览器 (undetected-chromedriver 优先，selenium-stealth 回退)
  → 应用反检测补丁 (覆盖 webdriver/plugins/chrome 等 JS 属性)
  → 初始化数据库 (CREATE TABLE + 自动迁移)
  → 遍历3个分类页 (影院热映/即将上映/家庭观影)

    → 遍历每部电影
      → parse_detail_page 解析详情页

        → 提取类型、导演、演员 (JS精准提取，清理CSS噪声)

反检测机制（四层防护）
Layer 1: undetected-chromedriver (最优先，专门绕过 Cloudflare)
Layer 2: selenium-stealth (回退方案，伪装浏览器指纹)
Layer 3: JS 属性覆盖 (webdriver, plugins, languages, chrome, permissions, devicePixelRatio)
Layer 4: 人类行为模拟 (随机滚动、随机延时、Cloudflare 检测等待)
